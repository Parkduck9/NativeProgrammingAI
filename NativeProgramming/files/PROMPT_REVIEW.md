# 프롬프트 검토 기준

AI가 작성한 코드 요청 프롬프트를 아래 기준으로 검토한다.

## 검토 항목

1. 한 번에 하나의 기능만 요청하는가?
2. 첫 번째 기능이 실행해서 확인 가능한가?
3. C++ 초보자가 이해할 수 있는 수준인가?
4. 코드 설명을 요구하고 있는가?
5. 실행 방법을 요구하고 있는가?
6. 검증 방법을 요구하고 있는가?
7. 발생 가능한 오류나 주의점을 요구하고 있는가?
8. 개선 방향을 질문하고 있는가?

## AI에게 요청할 검토 프롬프트

아래 프롬프트 초안을 위 기준에 따라 검토해줘.

검토할 때는 다음 형식으로 답변해줘.

1. 항목별 평가
2. 부족한 점
3. 수정 방향
4. 개선된 최종 프롬프트

# 1단계 코드 요청 프롬프트

원본 프롬프트를 검토 기준으로 평가한 결과,
범위가 너무 넓다는 문제를 확인하여 3개로 분리했다.

## [1차 프롬프트] 설계 토론 전용

```
당신은 C++ 초보자를 가르치는 시니어 멘토입니다.

아래 목표를 가진 콘솔 채팅 프로그램의 1단계 구조를 설계해 주세요.
아직 코드는 작성하지 마세요.

목표:
- 콘솔에서 사용자 입력을 받고 AI 응답을 출력한다
- "exit" 입력 시 종료한다
- 대화를 vector<Message>에 저장한다
- 나중에 OllamaChatModel로 교체할 수 있는 구조여야 한다

답변 형식:
1. 필요한 파일 목록과 각 파일의 역할
2. DummyChatModel만 사용 vs IChatModel 인터페이스 분리
   두 방안의 장단점 비교 및 추천 이유
3. 추천 구조의 클래스 관계도 (텍스트)
```

## [2차 프롬프트] 코드 작성 전용

```
앞서 토론한 구조대로 코드를 작성해 주세요.

환경: Windows 11, Visual Studio 2022, C++17

작성 조건:
- 초보자가 읽기 쉽게 작성한다
- unique_ptr, 가상 함수 등 생소한 문법에는 반드시 주석으로 이유를 설명한다
- 파일 순서:
  main.cpp → Message.h → ChatApp.h → ChatApp.cpp
  → DummyChatModel.h → DummyChatModel.cpp

코드 작성 후 반드시 포함할 내용:
1. Visual Studio에서 빌드하고 실행하는 방법 (단계별)
2. 정상 동작 확인 방법 (어떤 입력으로 테스트하면 되는가)
3. Windows에서 발생할 수 있는 오류와 해결 방법 (특히 한글 깨짐)
```

## [3차 프롬프트] 학습 정리 & 개선 전용

```
위에서 작성한 코드를 바탕으로 학습 내용을 정리해 주세요.

1. 이번 단계에서 배운 C++ 개념 목록과 한 줄 설명
2. 직접 수정해 보면 좋은 부분 3가지 (난이도 순)
3. 지금 당장 개선할 수 있는 점 (큰 리팩터링 없이 고칠 수 있는 것)
4. 2단계에서 자연스럽게 이어질 기능 제안
```

---

## 사용 순서

```
[1차] 설계 토론 프롬프트 전송
    ↓ AI 답변 확인 & 구조 동의
[2차] 코드 작성 프롬프트 전송
    ↓ 코드 받아서 빌드 + 실행 + 확인
[3차] 학습 정리 프롬프트 전송
    ↓ 정리 & 2단계 준비
```

## 원본 프롬프트 검토 결과 (참고)

| # | 항목 | 평가 | 이유 |
|---|---|---|---|
| 1 | 한 번에 하나의 기능만 요청하는가? | ❌ 미흡 | 구현 목표 8개 + 설계 토론 + 코드 + 학습 설명을 한 번에 요청 |
| 2 | 첫 번째 기능이 실행해서 확인 가능한가? | ⚠️ 부분 | 출력 예시는 있지만 어느 기능을 먼저 확인할지 불명확 |
| 3 | C++ 초보자가 이해할 수 있는 수준인가? | ✅ 충족 | 초보자 배려 원칙, 생소한 문법 주석 요구 명시 |
| 4 | 코드 설명을 요구하고 있는가? | ✅ 충족 | 설명 방식 섹션에서 구조 선택 이유 명시 |
| 5 | 실행 방법을 요구하고 있는가? | ❌ 없음 | 빌드/실행 방법 요청 없음 |
| 6 | 검증 방법을 요구하고 있는가? | ❌ 없음 | 동작 확인 방법 요청 없음 |
| 7 | 발생 가능한 오류나 주의점을 요구하고 있는가? | ❌ 없음 | 한글 깨짐 등 환경 오류 언급 없음 |
| 8 | 개선 방향을 질문하고 있는가? | ⚠️ 부분 | 다음 단계 제안은 있지만 현재 코드 개선 방향은 아님 |
